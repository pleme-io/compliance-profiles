Aggregation of government/community InSpec compliance profiles with BLAKE3 integrity
verification and profile indexing. Designed to feed the inspec-rspec transpiler.

## Architecture

```
compliance-profiles/
  src/
    lib.rs        -- module declarations + known_profiles()
    main.rs       -- CLI: list, hash, index, stats
    profile.rs    -- ProfileManifest, ProfileEntry, ProfileSource types
    index.rs      -- ProfileIndex: add/remove/find/save/load
    hasher.rs     -- BLAKE3 directory + file hashing
    sync.rs       -- Upstream sync logic (git submodule management)
    error.rs      -- Error types
  samples/        -- Sample InSpec controls for testing
  tests/          -- Integration tests
```

## Key types

- `ProfileEntry` -- a single compliance profile with metadata, hash, NIST/CIS coverage
- `ProfileIndex` -- collection of entries with stats, save/load to JSON
- `ProfileSource` -- enum: Mitre, DevSec, Gcp, Cis, Custom, AutoGenerated

## Commands

```bash
cargo run -- list                    # List all known profiles
cargo run -- list --provider aws     # Filter by provider
cargo run -- hash samples/linux-baseline  # BLAKE3 hash a profile dir
cargo run -- index samples -o index.json  # Generate index from profiles
cargo run -- stats                   # Show statistics
```

## Conventions

- Rust edition 2024, rust-version 1.89.0, MIT
- clippy pedantic enabled
- BLAKE3 for all hashing (standalone, no tameshi dependency)
- Sample InSpec controls must be realistic enough for inspec-rspec transpilation
